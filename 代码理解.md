## 一、状态来源

本项目涉及 4 类核心状态，其存储与读取方式如下：

### 1. 木材档案（木材档案库）

- **存储位置**：localStorage
- **存储 Key**：`woodPlanerWoodLibrary`，由常量 [WOOD_LIB_KEY](file:///Users/aj/Downloads/AI/0606/261/app.js#L22-L22) 定义
- **读取函数**：[loadWoodLibrary](file:///Users/aj/Downloads/AI/0606/261/app.js#L537-L541) —— JSON.parse 读取，失败或为空返回 `[]`
- **写入函数**：[saveWoodLibrary](file:///Users/aj/Downloads/AI/0606/261/app.js#L543-L545) —— 直接 JSON.stringify 后写入
- **数据结构**（每条记录）：`{ id, name, hardness, planerType, angleMin, angleMax, depth, note, createdAt, updatedAt }`
- **首次读取时机**：[init](file:///Users/aj/Downloads/AI/0606/261/app.js#L1142-L1155) 中调用 `renderWoodLibrary()`

### 2. 参数预设

- **存储位置**：localStorage
- **存储 Key**：`woodPlanerPresets`，由常量 [STORAGE_KEY](file:///Users/aj/Downloads/AI/0606/261/app.js#L21-L21) 定义
- **读取函数**：[loadPresets](file:///Users/aj/Downloads/AI/0606/261/app.js#L387-L391) —— JSON.parse 读取，失败或为空返回 `[]`
- **写入函数**：[savePresets](file:///Users/aj/Downloads/AI/0606/261/app.js#L393-L395)
- **数据结构**（每条记录）：`{ id, name, planerType, bladeAngle, cutDepth, woodHardness, woodName, note, createdAt }`
- **首次读取时机**：[init](file:///Users/aj/Downloads/AI/0606/261/app.js#L1142-L1155) 中调用 `renderPresets()`

### 3. 当前计算参数

- **存储位置**：仅存在于内存 `state` 对象（[app.js 第37-48行](file:///Users/aj/Downloads/AI/0606/261/app.js#L37-L48)），**没有持久化到 localStorage**
- **字段包括**：`planerType, bladeAngle, cutDepth, woodHardness, reverseMode, targetRa, compareOpen, compareSelA, compareSelB, currentWoodName`
- **来源**：
  - 默认值在 state 对象初始化时硬编码
  - 通过 `applyWoodToCurrent` 从木材档案导入
  - 通过 `loadPresetById` 从预设导入
  - 用户通过 slider、input、seg-btn、planer-tab 等 UI 组件实时修改

### 4. 对比面板状态

- **常量声明但未使用**：`COMPARE_STATE_KEY = 'woodPlanerCompareState'` 在 [app.js 第23行](file:///Users/aj/Downloads/AI/0606/261/app.js#L23-L23) 被定义，但**整个代码库从未使用它存取 localStorage**
- **实际存储位置**：仅存在于内存 `state` 对象的三个字段：
  - `state.compareOpen` —— 面板是否打开
  - `state.compareSelA` —— 方案 A 来源标识（`'current'` / `'wood:xxx'` / `'preset:xxx'`）
  - `state.compareSelB` —— 方案 B 来源标识
- **结论**：刷新页面后对比面板状态完全丢失，不会被持久化。

## 二、交互链路

### 链路 1：点击"带入参数"

1. **触发点**：木材档案卡片上的 `带入参数` 按钮，在 [renderWoodLibrary](file:///Users/aj/Downloads/AI/0606/261/app.js#L584-L594) 中通过事件委托绑定，`data-action="apply"`
2. **入口函数**：[applyWoodToCurrent(id)](file:///Users/aj/Downloads/AI/0606/261/app.js#L599-L634)
3. **数据流转**：
   - `loadWoodLibrary()` 读取全部木材档案 → find 找到目标记录 `w`
   - 将 `w.planerType / w.hardness / w.depth` 直接写入 state
   - **关键转换**：`bladeAngle` 不取档案原值，而是取 `(w.angleMin + w.angleMax) / 2`，再用 `clamp()` 限制在当前刨刀类型的推荐范围内
   - `state.currentWoodName = w.name`
4. **UI 同步**：更新 planer tabs、angle-slider、depth-slider、depth-input、hardness seg-btn 的 DOM 状态
5. **渲染出口**：`updateUI()`（主计算结果 + SVG），如对比面板开着还会调用 `renderCompare()`

### 链路 2：点击"对比A" / "对比B"

1. **触发点**：木材档案卡片上的按钮，`data-action="compareA"` 或 `compareB`，同样在 [renderWoodLibrary](file:///Users/aj/Downloads/AI/0606/261/app.js#L589-L590) 绑定
2. **入口函数**：[setCompareSelection(side, value)](file:///Users/aj/Downloads/AI/0606/261/app.js#L759-L773)
3. **数据流转**：
   - 将 `wood:${id}` 格式的标识写入 `state.compareSelA` 或 `state.compareSelB`
   - 同步设置 `<select>` 下拉框的 DOM value
   - 若面板未打开则打开（`state.compareOpen = true` + `display:block`）
4. **渲染出口**：[renderCompare()](file:///Users/aj/Downloads/AI/0606/261/app.js#L861-L919)

### renderCompare 内部链路

1. `refreshCompareSelects()` —— 重新从 localStorage 读取木材档案和预设，重建两个 `<select>` 的 `<option>`
2. `resolveCompareSource(state.compareSelA)` 和 `resolveCompareSource(state.compareSelB)` —— 根据来源标识解析参数
   - `'current'`：直接取内存 state
   - `'wood:xxx'`：从 `loadWoodLibrary()` 找记录，**同样取 angleMin/angleMax 平均值作为 bladeAngle**，并 clamp
   - `'preset:xxx'`：从 `loadPresets()` 找记录，直接用保存的 bladeAngle
3. `computeForward()` 分别计算 A、B 两组的切屑厚度、刨削力、粗糙度
4. `setCell()` 逐格渲染对比表格，差异项加 `.diff` class
5. `buildAdvice()` 分别生成 A、B 的加工建议列表

### 链路 3：点击"导出维护清单"

1. **触发点**：顶栏 `#btn-export` 按钮，在 [initExport](file:///Users/aj/Downloads/AI/0606/261/app.js#L922-L1027) 中绑定
2. **数据流转**（分三段可选拼接）：
   - **段 A（可选）**：若 `state.compareOpen` 为 true → 通过 `resolveCompareSource()` 取 A、B → `computeForward()` 实时重算两组结果 → `buildAdvice()` 生成建议 → 以"参数对比结果"为标题写入 CSV 行
   - **段 B**：若木材档案库非空 → `loadWoodLibrary()` 取全部 → 逐条展开为"木材档案库" CSV 行（展示原始 angleMin/angleMax，不取平均值）
   - **段 C**：若预设库非空 → `loadPresets()` 取全部 → 逐条 `computeForward()` **实时重算**切屑/力/Ra → 展开为"参数预设" CSV 行
3. **导出方式**：拼 CSV 字符串（`\uFEFF` BOM + `\r\n` 换行）→ `new Blob` → `URL.createObjectURL` → 动态创建 `<a download>` 触发下载 → 文件名 `刨刀维护清单_YYYYMMDD.csv`

### 链路 4：点击"打印角度对照卡"

1. **触发点**：顶栏 `#btn-print` 按钮，在 [initPrint](file:///Users/aj/Downloads/AI/0606/261/app.js#L1030-L1038) 中绑定
2. **数据流转**（4 个构建函数依次执行，均写入仅打印可见的 DOM）：
   - [buildPrintTable](file:///Users/aj/Downloads/AI/0606/261/app.js#L1040-L1055) → 读取内存常量 `REFERENCE_TABLE`（8 种木材 × 5 种刨刀的静态角度范围表）→ 渲染到 `#angle-table`
   - [buildPrintWoodLibrary](file:///Users/aj/Downloads/AI/0606/261/app.js#L1057-L1076) → `loadWoodLibrary()` 读取 → 渲染到 `#wood-print-table`
   - [buildPrintCurrentWood](file:///Users/aj/Downloads/AI/0606/261/app.js#L1078-L1095) → 从内存 `state` 读当前参数 + `computeForward()` 实时计算 → 渲染到 `#print-current-wood`
   - [buildPrintCompare](file:///Users/aj/Downloads/AI/0606/261/app.js#L1097-L1139) → 若 `state.compareOpen` 则 `resolveCompareSource()` 解析 + `computeForward()` 计算 → 渲染到 `#print-compare`
3. **打印触发**：`setTimeout(() => window.print(), 200)` —— 给 DOM 渲染留 200ms
4. **CSS 协同**：[styles.css @media print](file:///Users/aj/Downloads/AI/0606/261/styles.css#L744-L793) 将 `.no-print` 的交互界面全部隐藏，仅显示 `.print-only` 的 `#print-card` 区域

## 四、潜在风险点（展示值 vs 实际计算值不一致）

### 风险点 1：木材档案的角度"范围展示" vs "平均值计算"

- **位置**：
  - 展示侧：[renderWoodLibrary 第567行](file:///Users/aj/Downloads/AI/0606/261/app.js#L567-L567) —— 卡片上展示 `${w.angleMin}°–${w.angleMax}°`
  - 计算侧 1：[applyWoodToCurrent 第605-606行](file:///Users/aj/Downloads/AI/0606/261/app.js#L605-L606) —— 实际带入 `(angleMin + angleMax) / 2`，且 clamp 到刨刀推荐范围
  - 计算侧 2：[resolveCompareSource 第819-820行](file:///Users/aj/Downloads/AI/0606/261/app.js#L819-L820) —— 对比面板中木材来源同样取平均值并 clamp
  - 打印侧：[buildPrintWoodLibrary 第1071行](file:///Users/aj/Downloads/AI/0606/261/app.js#L1071-L1071) 打印的是原始范围，而 [buildPrintCompare 第1126/1133行](file:///Users/aj/Downloads/AI/0606/261/app.js#L1126-L1133) 打印的是平均值
- **现象**：用户看到卡片上写"角度 30°–40°"，但带入后实际用的是 35°（或被 clamp 后的值），卡片本身不会显示实际计算值；打印木材档案表和打印对比结果中的角度数值口径不同。

### 风险点 2：`state.currentWoodName` 与实际参数可能不同步

- **位置**：
  - 设置 currentWoodName 的两处：[applyWoodToCurrent 第609行](file:///Users/aj/Downloads/AI/0606/261/app.js#L609-L609)、[保存预设时第529行](file:///Users/aj/Downloads/AI/0606/261/app.js#L529-L529)
  - 消费侧：[resolveCompareSource 第808行](file:///Users/aj/Downloads/AI/0606/261/app.js#L808-L808)（对比面板"当前参数"显示的木材名）、[buildPrintCurrentWood 第1091行](file:///Users/aj/Downloads/AI/0606/261/app.js#L1091-L1091)（打印当前加工木材）
- **现象**：用户从"紫檀"木材档案带入参数后，手动把硬度从"极硬"改成"软木"、角度改成 20°，此时 `state.currentWoodName` 仍然是"紫檀"——对比面板显示"当前参数 · 紫檀"，打印时也写"紫檀"，但实际计算用的是软木 K=2.0 和 20° 的数据，木材名与计算参数严重脱节。

### 风险点 3：`animateValue` 动画期间展示值 ≠ state 真实值

- **位置**：[updateUI 第234行](file:///Users/aj/Downloads/AI/0606/261/app.js#L234-L234) 调用 `animateValue(document.getElementById('angle-value'), state.bladeAngle, ...)`，动画实现见 [animateValue 第215-228行](file:///Users/aj/Downloads/AI/0606/261/app.js#L215-L228)
- **现象**：300ms 动画过渡期间，DOM 上显示的角度是插值（如从 30 过渡到 35 的过程中会显示 31.2、32.7 等），而 `state.bladeAngle` 已经是 35，切屑/力/Ra 也已按 35 重新计算。用户在这 300ms 窗口内看到的角度值与其他三个计算结果不匹配；若恰在此时截图或手动读取 DOM，会拿到不一致的展示值。

### 风险点 4：导出 CSV 时预设的计算值并非"保存时的值"

- **位置**：[initExport 第994-998行](file:///Users/aj/Downloads/AI/0606/261/app.js#L994-L998) —— 导出预设时 `computeForward(p.bladeAngle, p.cutDepth, hd ? hd.K : 4.5)` 实时计算切屑/力/Ra
- **现象**：预设保存时并未保存计算结果，CSV 中导出的切屑厚度、刨削力、Ra 是导出当下用当前代码计算的。如果未来 `computeForward` 公式、`WOOD_HARDNESS` 中 K 值或 `BLADE_WIDTH` 常量有调整，旧预设导出的数值会和保存时页面上看到的不一样。

### 风险点 5：COMPARE_STATE_KEY 声明但未持久化

- **位置**：[app.js 第23行](file:///Users/aj/Downloads/AI/0606/261/app.js#L23-L23) 声明了 `COMPARE_STATE_KEY`，但全代码库 grep 不到任何使用它读写 localStorage 的地方。
- **现象**：对比面板的开关状态、A/B 选择仅在内存 state 中。用户打开对比面板、选好两组参数后刷新页面，所有对比状态消失——用户预期对比状态会和木材档案/预设一样被保存，实际并不会。

### 风险点 6：打印对比时若所选来源已被删除，静默不打印

- **位置**：[buildPrintCompare 第1099-1110行](file:///Users/aj/Downloads/AI/0606/261/app.js#L1099-L1110)
- **现象**：`state.compareOpen` 为 true，但 A 或 B 指向的木材/预设已被删除时，`resolveCompareSource` 返回 null → 代码将 `#print-compare` 清空并移除 className，不打印对比，也不给出任何警告。此时界面上对比面板可能仍处于打开状态，但里面是空的，打印出来也没有对比段落——用户看到界面有对比面板，打印预览却缺失内容。

### 风险点 7：导出 CSV 中对比段的"角度/深度"口径与木材档案段不一致

- **位置**：
  - 对比段角度：[initExport 第954行](file:///Users/aj/Downloads/AI/0606/261/app.js#L954-L954) 写的是 `a.bladeAngle`（木材来源时是取过平均并 clamp 的单一数值）
  - 木材档案段角度：[initExport 第981-982行](file:///Users/aj/Downloads/AI/0606/261/app.js#L981-L982) 写的是 `w.angleMin` 和 `w.angleMax` 两个独立列
- **现象**：同一份木材档案，在 CSV 的"参数对比结果"段表现为单一角度值，在"木材档案库"段表现为角度下限+上限两列。两份数据在同一份导出文件中口径不同，用户对比查看时容易混淆。
